## 三、管线设计（先分析结构，再落地，契合大帅爬虫规范）

```
[1] 登记候选源  → kb-sources.json 追加 GAP_CATALOG 项（type=web, ingest=未实现）
[2] 结构分析    → 对目标 URL 做 HEAD/GET，解析列表页与详情页 DOM 结构（不抓正文）
[3] 认证接入    → 大帅提供 Cookie/Token 注入 fetch 头（.env: NHC_AUTH_COOKIE 等）
[4] 正文抓取    → 按分析出的选择器抽取指南正文，存临时区（人工抽检 1~2 份核对权威度）
[5] 结构化      → build-guide-index / extract-outline / extract-entities 复用现有管线
[6] 入库登记    → 写入 medical-knowlegde-base/*.md + 刷新 kb-sources 指纹
[7] 覆盖复验    → kb-update coverage 确认偏科指数下降、缺口专科闭合
```

## 四、防杜撰与合规约束（强制）

- 抓取失败/无认证 → 该候选源 `ingest` 保持「未实现」，绝不生成占位正文。
- 入库前**逐份人工抽检**：标题、发布机构、年份须与官方一致；不一致整份丢弃重抓。
- 正文一律标注来源 URL 与抓取日期于文首元数据，便于溯源与过期刷新。
- 仍受 System Prompt 约束：知识库外问题明确说明超出范围，不臆造。

## 六、官方文件直供操作手册（大帅投文件即跑）

1. 大帅在已登录浏览器打开目标指南，另存为 **PDF / 网页 HTML / Markdown**。
2. 把文件丢入任意目录（建议 `medical-knowlegde-base/_inbox/`）。
3. 臣执行（或授权后自动）：
   ```
   node scripts/ingest-raw.mjs <文件路径> --name "指南标准名(含年份)" --dept <专科> --src "<来源URL>"
   ```
   - `--dept` 可省略，脚本按名称自动归类（儿科/急诊/产科等优先级置顶）。
   - 示例：`node scripts/ingest-raw.mjs ./_inbox/儿童急救指南.pdf --name "儿童急危重症诊疗指南（2024版）" --dept 儿科 --src "https://www.nhc.gov.cn/..."`
4. 脚本自动：规范化 MD 入库 → 登记 kb-sources → 刷新大纲 `.outline.json`。
5. 大帅**人工抽检**标题/机构/年份与官方一致后，`kb-update coverage` 验偏科指数下降。
