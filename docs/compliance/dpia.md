# 数据保护影响评估（DPIA · 简化版）

> 依据：GDPR Art.35 类义务的精神，对「医疗 RAG 问答系统处理个人健康信息（PHI）」的处理活动做简化影响评估。
> 本评估锚定代码事实（见同目录其他文档），非形式合规文书，供内部风险自评与验收。

## 1. 处理活动描述

| 项 | 内容 |
|---|---|
| 系统 | 医疗 Agentic RAG（「医典」）——基于官方临床指南的循证问答助手 |
| 处理目的 | 辅助临床/健康信息查询，提供带溯源的指南引用答案 |
| 处理方式 | 检索增强生成（RAG）+ 多道安全护栏 + 患者画像上下文注入 |
| 数据主体 | 问诊中提及的患者（含用户本人或他人） |
| 数据类别 | PHI：年龄/性别/过敏史/既往病史/当前用药；PII：对话中的手机/身份证/邮箱 |

## 2.  necessity 与相称性（Proportionality）

- **相称**：画像仅存结构化最小字段（无自由文本叙述），目的限于「关键医学事实不丢失（尤其过敏禁忌）」，不过度收集。
- **必要**：抗 Pi compaction 摘要丢失而设计，是医疗安全（防过敏致死性推荐）的合理必要手段。
- **替代**：未采用更侵入的全程录音/全量留存，符合数据最小化。

## 3. 风险评估

| 风险 | 可能性 | 严重性 | 缓释措施 | 残余 |
|---|---|---|---|---|
| 未授权访问本地密文 | 低 | 高 | AES-256-GCM 认证加密 + 密钥 `chmod 600` + gitignore 不入仓 | 低 |
| 明文残留磁盘 | 极低 | 高 | 明文绝不落盘；旧明文透明迁移为密文 | 极低 |
| 审计篡改 / 抵赖 | 低 | 中 | HMAC-SHA256 防篡改哈希链（`audit-chain.mjs`） | 低 |
| LLM 提供方见 PHI | 中 | 中 | TLS + 输入脱敏 + 最小字段 + 免费档不训练 | 中（见 §5） |
| 越界/有害输出 | 低 | 高 | scope-guard 硬阻断 + faithfulness/conflict 真阻断（E1） + 六维护栏断言 | 低 |
| 命令级卡死/越权 | 低 | 高 | bash-guard 拦截 + 超时夹紧（P0） | 低 |
| 密钥泄露 | 低 | 高 | 环境变量零信任（不硬编码）+ gitignore | 低 |
| 对话 PHI 入库泄露 | 极低 | 高 | `.pi/sessions/` 等 gitignore 绝对不入仓 | 极低 |

## 4. 缓释措施总览（已落地）

- **加密**：AES-256-GCM 静态加密（密钥零信任）。
- **脱敏**：输入 PII 掩码出境前。
- **审计**：HMAC 哈希链不可抵赖，仅记元数据不记原值。
- **护栏**：5 道分层（越界/命令/忠实度/冲突/脱敏），代码层硬阻断 + 运行时真生效。
- **被遗忘权**：`forget_patient` 安全擦除（覆写+删除）。
- **数据红线**：含 PHI/密钥产物一律 gitignore。

## 5. 残余风险与约束条件

- **LLM 推理可见**：PHI 在推理会话中对提供方（sensenova）客观可见。约束：仅结构化最小字段 + 脱敏 + 不训练；建议生产化前补签 DPA（见 `third-party.md` §5）。
- **密钥未轮换**：单版本静态密钥，缺定期轮换接口（已知缺口）。
- **无自动过期**：保留期依赖「按需擦除」，未设 TTL 自动销毁。
- **自由文本 PHI**：用户自由叙述中的未结构化 PHI 不自动脱敏，依赖用户自决与越界护栏。

## 6. 结论

**残余风险等级：LOW（低）**，在以下约束条件下成立：
1. 仅使用免费额度 LLM 且不训练条款；
2. 密钥经环境变量管理、不入库；
3. 含 PHI 产物严格 gitignore；
4. 用户可经 `forget_patient` 行使被遗忘权。

> 若系统进入生产化（真实患者数据、付费模型、跨境传输），须重评并补：DPA 签署、密钥轮换、保留期 TTL、自助访问/导出工具。
> 评估随代码演进复审；本文档与代码冲突时以代码为准。
