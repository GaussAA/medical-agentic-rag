纠错后：deepseek 内置复原
Provider模型上下文Max-Out思考来源

deepseekdeepseek-v4-pro1M384Kyes内置
agnesagnes-2.0-flash256K65.5Kno扩展注册
sensenovasensenova-6.7-flash-lite256K65.5Kyes扩展注册

纠错沉淀：deepseek 无需自写 Provider 扩展

大帅指正——deepseek 是 Pi 内置 Provider，随启动经 loadBuiltInModels 自动播种。曾误补的 deepseek-provider.ts 因同名 registerProvider 覆盖内置，致 max-out 由 384K 跌至 65.5K、thinking 变 no、丢 deepseek-v4-pro 模型。已删除该扩展，故障转移表仅保留探测元数据（baseUrl+authEnv）。

✓ 实证：删除后 --list-models 显示 384K / thinking=yes / pro 模型复现
